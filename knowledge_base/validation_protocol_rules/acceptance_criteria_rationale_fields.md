# Acceptance Criteria Rationale Fields

## 1. Purpose

This file defines the minimum rationale fields required when drafting analytical method validation acceptance criteria.

Acceptance criteria generated by the platform are draft suggestions only. They must be reviewed and approved by the responsible analytical lead, QA, RA, and/or method owner before use.

---

## 2. Required Rationale Fields

Each acceptance criterion should include the following fields.

| Field | Required | Description |
|---|---:|---|
| validation characteristic | Yes | Specificity, accuracy, precision, range, robustness, etc. |
| proposed criterion | Yes | Draft acceptance criterion or decision rule |
| intended purpose link | Yes | How the criterion supports release, stability, transfer, characterization, etc. |
| reportable result link | Yes | How the criterion supports the final reported result |
| method type dependency | Yes | Why the criterion is suitable for this method type |
| sample matrix dependency | Conditional | Whether matrix affects the criterion |
| source of criterion | Yes | Development data, prior method, internal rule, official source, literature, vendor method, AI-inferred |
| evidence level | Yes | Evidence classification |
| confidence | Yes | High, medium, low, or not assessed |
| data required to confirm | Yes | What data are needed before finalization |
| human review required | Yes | Must normally be Yes |

---

## 3. Source of Criteria

Possible sources include:

```text
- official regulatory source
- approved internal rule
- prior validated method
- method development data
- historical internal case
- vendor / laboratory-provided method proposal
- curated scientific literature
- AI-inferred suggestion
- human review required
```

If a criterion is AI-inferred or based only on a generic rule, it should be marked as low or medium confidence and require review.

---

## 4. Drafting Rules

1. Do not present draft criteria as approved criteria.
2. Do not invent numerical limits without source, data, or explicit user-provided rationale.
3. If criteria depend on method development data, state that data are required.
4. If criteria are derived from an external lab proposal, label the source as vendor / laboratory-provided information.
5. If criteria are intended for GxP, release, stability, vendor-facing, or regulatory use, human review is required.
6. If method variability is expected to be high, flag the criterion for SME review.
7. If the criterion does not clearly support the intended reportable result, mark it as a gap or missing rationale.

---

## 5. Characteristic-Specific Rationale Prompts

| Characteristic | Rationale Prompts |
|---|---|
| Specificity / selectivity | What interference is being excluded? What matrix, impurity, degradation product, or related analyte is relevant? |
| Accuracy | What reference, spike, recovery, or comparison supports expected value? |
| Precision | What variability is acceptable for the intended use? What runs, analysts, instruments, or days are included? |
| Linearity / calibration model | What model is used? Is the model appropriate for the response? What range is covered? |
| Range / reportable range | Does the range cover specification, expected sample concentration, and dilution scheme? |
| LOD / LOQ | Why is detection or quantitation at low level needed? What reporting threshold is relevant? |
| Robustness | Which method parameters are critical? What variation is scientifically justified? |
| System suitability | Does the SST monitor critical method performance, not just generic instrument performance? |
| Matrix effect | Does the actual sample matrix alter response, recovery, curve behavior, or detection? |
| Parallelism | Does sample dilution behavior match reference standard behavior where applicable? |

---

## 6. Required Output Format

A generated protocol should represent acceptance criteria as a table.

| Validation Characteristic | Proposed Criterion | Rationale | Evidence Level | Confidence | Data Needed | Human Review |
|---|---|---|---|---|---|---|

---

## 7. Human Review Statement

Acceptance criteria are not final until reviewed and approved through the appropriate internal process.
