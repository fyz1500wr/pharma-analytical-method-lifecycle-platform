# Method Principle Cards

## Purpose

This directory stores reviewer-friendly technical principle cards for analytical technologies.

These cards are intended to help PMs, reviewers, QA, RA, and analytical leads rapidly understand unfamiliar or complex analytical reports.

## Card Should Explain

- What the method is
- What it measures
- How the signal is generated
- What conclusions it can support
- What conclusions it cannot support
- Typical controls
- Key data to inspect
- Common limitations
- Common review questions
- SME review triggers

## Important Rule

A principle card is educational and contextual. It should not be used alone to approve a method or validation report.
