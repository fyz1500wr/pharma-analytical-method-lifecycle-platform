# Cell-Based Potency Assay — Method Principle Card

## 1. Technology Status

```text
Known but method-specific mapping required
```

## 2. Principle Summary

A cell-based potency assay measures a biological response generated by cells after exposure to a reference standard and test sample.

The response may be used as a surrogate for product biological activity when the response is scientifically linked to the intended product mechanism or quality attribute.

## 3. What It Measures

A cell-based potency assay generally measures a functional biological response, not product identity, purity, or protein content.

The reportable result may be relative potency, activity, response percentage, inhibition, stimulation, or another response model depending on assay design.

## 4. Signal Source

The signal usually comes from a cell response such as:

- viability;
- reporter signal;
- proliferation;
- inhibition;
- biomarker response;
- cytopathic effect protection;
- enzyme or luminescent response;
- other product-specific biological readout.

## 5. Suitable For

Potentially suitable for:

- potency assessment;
- stability trend support;
- comparability support;
- biological activity characterization;
- method development or bridging studies.

Suitability depends on method design, validation status, reference standard strategy, and product-specific rationale.

## 6. Not Sufficient For

A cell-based potency assay is generally not sufficient by itself for:

- product identity;
- purity;
- impurity quantitation;
- protein content;
- degradation product identification;
- structural characterization.

Separate orthogonal methods may be needed.

## 7. Key Controls

Common controls include:

- reference standard;
- working standard;
- positive control;
- negative control;
- matrix control;
- cell control;
- assay control;
- blank or vehicle control;
- system suitability or run validity controls.

## 8. Reviewer Focus

Reviewers should inspect:

1. intended purpose;
2. biological relevance of the response;
3. reference standard strategy;
4. curve model and range;
5. matrix effect;
6. parallelism or dilution-response comparability;
7. run validity and system suitability criteria;
8. failed-run and repeat-run handling;
9. precision and intermediate precision design;
10. sample and reagent stability;
11. whether the method supports release, stability, transfer, or only development use.

## 9. Common Failure Modes

- assay response not clearly linked to potency;
- matrix effect not evaluated;
- parallelism not assessed;
- reference standard strategy unclear;
- curve model not justified;
- system suitability criteria too generic;
- high variability without sufficient run design;
- failed runs excluded without predefined rule;
- report conclusion broader than supporting data;
- method claimed suitable for release without adequate validation scope.

## 10. SME Review Triggers

SME review is recommended when:

- method supports release or stability;
- method is cell-based or high-variability;
- matrix effect is possible;
- relative potency is calculated;
- reference standard strategy is unclear;
- curve fitting or parallelism is central to the result;
- report is vendor-generated and raw data are limited;
- the method is new, transferred, or not internally mapped.
