# Dashboard Design

## Purpose

This directory will contain dashboard design notes and later dashboard implementation files.

Dashboard should provide management and review visibility, not replace GitHub as the source of truth.

## Planned Dashboard Views

1. Product-method mapping status
2. Validation protocol generation status
3. Analytical report review queue
4. Technical context card coverage
5. Regulatory source refresh status
6. Evidence pack status
7. Open gaps and reviewer questions
8. SME review required items
9. Vendor / lab recurring issue trends

## Notion Integration Principle

Notion may be used as a dashboard or task tracking layer. It should link back to GitHub evidence, templates, issues, and outputs rather than duplicating core rules.
