# Technical Context-Aware Analytical Report Review Workflow

## Purpose

Help reviewers rapidly understand unfamiliar, complex, or emerging analytical reports and identify technical, regulatory, and documentation gaps.

This workflow is a core child workflow of the Pharmaceutical Analytical Method Lifecycle Intelligence Platform.

## Inputs

- Analytical report, protocol, transfer report, or vendor proposal
- Product profile, if available
- Analytical method profile, if available
- Analytical technology registry
- Method principle cards
- Validation gap rules
- Reviewer question bank
- Evidence packs

## Steps

1. Identify document type.
2. Extract method name, method type, sample matrix, intended purpose, reportable result, controls, standards, validation characteristics, and conclusions.
3. Determine technology status:
   - Known and internally mapped
   - Known but not internally mapped
   - Emerging / insufficient internal knowledge
   - Unclear terminology / requires clarification
4. Generate or retrieve a technical principle card.
5. Assess method suitability for the stated analytical purpose.
6. Review technical design and documentation completeness.
7. Apply validation gap rules where applicable.
8. Generate critical / major / minor / clarification gaps.
9. Generate reviewer questions.
10. Mark SME review triggers.
11. Attach evidence-level and confidence labels.
12. Add human review statement.

## Outputs

- Technical principle card
- Method suitability assessment
- Gap assessment
- Reviewer question list
- SME review flag
- Evidence-level labels
- Human review statement

## Important Rule

This workflow does not approve reports. It helps reviewers understand, triage, and ask better questions.
