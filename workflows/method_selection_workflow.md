# Analytical Method Selection Workflow

## Purpose

Recommend analytical test categories and candidate method types based on product profile, analytical purpose, lifecycle stage, and regulatory context.

## Inputs

- Product profile JSON
- Product-type / method matrix
- Regulatory source priority matrix
- Known CQAs
- Sample type and matrix
- Target regions

## Steps

1. Read product profile.
2. Identify required analytical purposes.
3. Match product type and sample type to candidate test categories.
4. Classify each suggested test as required, conditional, or optional.
5. Assign candidate method types.
6. Identify key method development concerns.
7. Identify validation implications.
8. Identify missing information and SME review needs.
9. Attach evidence level and confidence labels.

## Outputs

- Recommended analytical categories
- Candidate method types
- Required / conditional / optional classification
- Rationale
- Missing information
- Risk flags
- Human review requirement

## Important Rule

This workflow recommends analytical directions. It does not approve final specifications, validation strategies, or regulatory filing positions.
