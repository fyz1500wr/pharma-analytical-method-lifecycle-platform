# Project Roadmap

## Phase 0 — Foundation

Status: In progress

Goals:

- Establish project direction and architecture principles.
- Establish basic folder structure.
- Establish initial schemas.
- Establish initial workflows and templates.
- Establish evidence and human review rules.

Key outputs:

- `PROJECT_DIRECTION_CALIBRATION.md`
- `PROJECT_INSTRUCTION.md`
- `PROJECT_STATE_CONTINUATION.md`
- `knowledge_base/`
- `schemas/`
- `workflows/`
- `templates/`
- `.github/workflows/schema_validation.yml`

## Phase 1 — Product → Method Matrix MVP

Initial scope:

```text
biologics / recombinant proteins / drug product release and stability
```

Goals:

- Build initial product-type / method matrix.
- Define required / conditional / optional analytical categories.
- Link method categories to evidence level and review flags.
- Generate sample product profile examples.

## Phase 2 — Validation Protocol Generator

Goals:

- Build validation characteristic rules.
- Build protocol draft generation logic.
- Add sample protocol outputs.
- Add schema-based validation for generated outputs.

## Phase 3 — Technical Context-Aware Report Review

Goals:

- Build technical context card examples.
- Build report gap assessment examples.
- Build reviewer question bank.
- Add unknown / emerging analytical technology mode.

## Phase 4 — Regulatory / Literature Intelligence

Goals:

- Add regulatory source refresh workflow.
- Add evidence pack schema and examples.
- Design MCP access layer.
- Add source update dashboard status.

## Phase 5 — Closed-Loop Knowledge Management

Goals:

- Capture recurring review gaps.
- Update method checklists and question bank.
- Track vendor / lab recurring issues.
- Feed review outcomes back into protocol and report templates.
