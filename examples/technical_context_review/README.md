# Technical Context Review Examples

## Purpose

This directory stores synthetic examples for the Technical Context-Aware Analytical Report Review Module.

These examples are designed to show how the platform should separate:

1. technical context;
2. method suitability assessment;
3. gap assessment;
4. reviewer questions;
5. SME review flags;
6. evidence-level labeling;
7. human review requirements.

## Important Rules

- Examples are synthetic and non-confidential.
- Examples do not approve any method or report.
- Examples must not include real product records, vendor reports, patient data, or controlled GxP documents.
- Any output intended for actual CMC, QA, RA, laboratory, vendor-facing, GxP, or regulatory use requires human review.
